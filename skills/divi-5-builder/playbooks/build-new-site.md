# Build New Divi 5 Site

Use this playbook when the user wants to plan or build a new Divi 5 website, especially when the request spans several pages, global templates, design variables, presets, menus, or launch checks.

## Table Of Contents

- Intake
- Build Order
- Page Assembly
- Verification

## Intake

Confirm the site can be built with native Divi 5 before proposing custom code:

- Business type, primary offer, audience, and conversion goal.
- Required pages, posts, products, forms, booking, memberships, or integrations.
- Brand inputs: logo, colors, fonts, imagery, sitemap, copy, CTAs, social links, business details.
- Whether the site is blank, partially built, cloned from another site, or replacing an existing site.
- Whether the user needs beginner coaching or independent execution.

If key content is missing, create a build-ready outline and use clear placeholders rather than blocking.

## Build Order

1. Set up the site baseline: WordPress, Divi 5, license, permalink structure, required plugins, staging/backups when applicable.
2. Create core pages first: Home, About, Services/Products, Contact, legal pages, and any required landing pages.
3. Define Design Variables for color, type, spacing, radius, logos, business text, CTAs, and links.
4. Create Option Group Presets for typography, buttons, spacing, images, cards, forms, and backgrounds.
5. Create Element Presets for common modules: headings, text, buttons, blurbs, images, forms, cards, and Woo modules if needed.
6. Build the global header and footer in Theme Builder.
7. Build the homepage, then inner pages, then blog/archive/search/product templates.
8. Add loops, menus, interactions, canvases, and dynamic content only where they serve the content.
9. QA desktop, tablet, phone, public frontend, forms, menus, accessibility basics, and performance.

## Page Assembly

Use the same page pattern repeatedly:

1. Add sections and rows first.
2. Add Groups when content should move or style as one unit.
3. Add modules with Admin Labels.
4. Apply variables and presets.
5. Fill real content or clear placeholders.
6. Use Flexbox for one-axis alignment and CSS Grid for two-axis layouts.
7. Use Loop Builder when repeated content should stay connected to WordPress data.
8. Use Interactions and Canvases after the static layout works.

Prefer stable, reusable section patterns: hero, trust strip, value grid, process/timeline, service cards, proof/testimonials, pricing, FAQ, CTA, contact.

## Verification

Before calling the site build complete:

- Variables and presets are used for repeated design decisions.
- Header/footer appear on intended pages and work at all breakpoints.
- Menus, buttons, forms, phone/email/social links, and dynamic links work.
- Images have appropriate alt text and do not distort at responsive sizes.
- Heading order, contrast, focus states, and tap targets are acceptable.
- Public frontend matches the builder closely after cache/static CSS refresh.
- Handoff notes identify variables, presets, templates, loops, canvases, and known dependencies.
