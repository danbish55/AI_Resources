# Client Handoff

Use this playbook when producing handoff notes, maintenance instructions, client training, or internal documentation for a Divi 5 site.

## Table Of Contents

- What To Document
- Client-Safe Editing
- Maintenance
- Handoff Format

## What To Document

Record:

- Important pages and Theme Builder templates.
- Global header/footer and any assigned body templates.
- Design Variables and which ones clients may edit.
- Presets used for typography, buttons, cards, forms, images, and Woo modules.
- Divi Library/Cloud items, loops, canvases, interactions, and menus.
- Forms, payment gateways, email providers, booking tools, memberships, cache, SEO, analytics, and security plugins.
- Known limitations, intentional workarounds, and areas that should be edited only on staging.

## Client-Safe Editing

Explain safe edits:

- Text, images, links, and simple module content.
- Blog posts, products, categories, menus, and media.
- Design Variables explicitly approved for client use, such as phone number, address, CTA text, or logo.

Warn before:

- Editing Theme Builder templates.
- Changing presets or variables that affect many pages.
- Deleting modules, rows, sections, loops, canvases, or menus.
- Adding global CSS, plugin updates, Divi updates, or bulk imports.

## Maintenance

Recommend:

- Backup before updates.
- Staging for major Divi, WordPress, WooCommerce, or plugin updates.
- Test forms, checkout, menus, header/footer, and key landing pages after updates.
- Check Divi 5 changelog/status before troubleshooting version-sensitive issues.
- Keep a record of custom code locations if any custom CSS/JS/PHP was approved.

## Handoff Format

Use concise Markdown:

1. Site map and template map.
2. Edit-safe areas.
3. Design system map.
4. Forms/integrations map.
5. WooCommerce notes if relevant.
6. Maintenance checklist.
7. Known risks or future improvements.
