## Fact Versus Hype

Facts to rely on:

- Divi 5 has official Help Center collections for getting started, migration, Visual Builder basics, Visual Builder core concepts, advanced concepts, modules, WooCommerce modules, option groups, features, Flexbox, CSS Grid, troubleshooting, Cloud, and email integrations.
- Elegant Themes' May 12, 2026 Divi 5.5 release notes say the beta phases have ended and Divi 5 update cadence is moving into weekly updates. Keep checking current status/changelog before answering time-sensitive availability questions.
- Divi 5 adds or expands concrete systems: Design Variables, Variable Generator, Option Group Presets, Element Presets, stacked/nested presets, Extend Attributes, Composable Settings, settings search/filtering, responsive editor, customizable breakpoints, Flexbox, CSS Grid, nested rows, nested modules, Group and Group Carousel modules, Interactions, Canvases, Canvas Portal, Loop Builder, Dynamic Content, Editable Theme Builder Areas, Page Manager, Preview Mode, Content Drill Down, semantic elements/custom HTML wrappers, Link and Dropdown modules, Menu Loops, WooCommerce modules, Divi Cloud support, and AI-assisted workflows.
- Many Divi 5 workflows are builder-native and do not require custom code when variables, presets, layout controls, interactions, or Theme Builder templates solve the problem.
- Migration and compatibility still deserve caution. Use staging for existing sites, check third-party module compatibility, and verify frontend rendering after conversion.

Marketing claims to treat carefully:

- "Faster", "game-changing", "most important", and "complete site editing" are useful directional claims, but they are not a substitute for testing the exact site, plugin stack, hosting, and workflow.
- "Ready for production" is now an official Divi 5.5 release-note claim, but production launches still require staging, plugin compatibility checks, cache/static CSS verification, responsive QA, and frontend review.
- Free design downloads and tutorial examples are inspiration and starting points, not proof that a pattern is right for the user's brand, content, accessibility needs, or performance budget.
- AI Agent capabilities can speed up builder operations, but destructive actions and site-wide changes still need human review.

## Feature Map

Design system:

- Design Variables store reusable colors, fonts, numbers, images, text, and links.
- Variable Generator creates color palettes and fluid sizing systems from builder controls.
- Relative Colors and HSL make related color roles easier to maintain.
- Advanced Units allow CSS functions and variables in many numeric settings.
- Presets should carry repeated styling. Use Option Group Presets for setting groups, Element Presets for complete modules/components, and stacked/nested presets for layered systems.
- Preset Manager and Preview systems help audit and select presets.
- Extend Attributes propagates selected settings, content, presets, or values across a defined scope.
- Composable Settings expose controls for module sub-elements that previously needed more custom CSS.
- Style Inspector helps identify where an element's style comes from.
- Image option presets allow shared image styling across images inside different Divi modules. Aspect ratio and framing settings help make inconsistent uploaded image dimensions fit repeatable card, loop, and editorial designs.

Layout:

- Sections, Rows, Columns, and Modules remain the basic mental model.
- Flexbox is best for one-dimensional alignment, reordering, wrapping, spacing, and responsive direction changes.
- CSS Grid is best for two-dimensional layouts, card grids, editorial layouts, sticky sidebars, nested grid compositions, and more precise placement.
- Nested Rows and Nested Modules allow richer component structures.
- Group Module is the container for grouped content; Group Carousel turns grouped cards into carousel patterns.
- Use admin labels and Layers whenever a page has nested structures, loops, interactions, portals, or several similar modules.

Theme Builder and dynamic content:

- Theme Builder handles global headers, footers, archive templates, search templates, single post/product templates, and site-wide layout logic.
- Editable Theme Builder Areas allow header, footer, and post template areas to be edited more directly while working in context.
- Dynamic Content connects module fields to WordPress data.
- Loop Builder builds repeated layouts from posts, taxonomies, custom post types, ACF fields, ACF repeater fields, and WooCommerce data.
- For blog/archive/search/product grids, prefer Theme Builder plus Loop Builder over manually duplicating cards.
- The Divi 5 Mastery Course Part 8 pattern uses Theme Builder templates for All Posts, Category Pages, Author Pages, Search Results, and 404 pages, with Dynamic Content for post title, featured image, author details, dates, author bio, post content, and archive context.

Menus and navigation:

- Menu Module remains useful for WordPress menus.
- Link Module and Dropdown Module make custom navigation structures more composable.
- Menu Loops support dynamic navigation and repeated menu structures.
- Interactions and Canvases can produce off-canvas mobile menus, mega menus, sidebars, popups, and reveal panels.

Interactions and canvases:

- Interactions are built from trigger, effect, and target. Use them for toggles, popups, hover/mouse effects, scroll behavior, and reveal patterns.
- Canvases hold off-page or reusable content such as popups, slide-ins, off-canvas nav, staged content, and reusable panels.
- Canvas Portal places canvas content into a page or component context.
- Always provide a non-hover path for touch users when hover is not enough.

Semantic and accessibility controls:

- Semantic elements and custom HTML wrappers can improve document structure when used intentionally.
- Use `header`, `footer`, `main`, `section`, `aside`, `figure`, and `figcaption` according to actual page meaning, not visual appearance alone.
- Prefer real links for navigation and real buttons for actions.
- Use ARIA roles sparingly and only to clarify behavior that semantic HTML does not already provide.

WooCommerce:

- Divi 5 has Help Center coverage for WooCommerce modules and blog guidance for product pages, category pages, cart/checkout, and product loops.
- Prefer product/category/archive templates with Theme Builder and dynamic Woo modules.
- Verify cart, checkout, focus states, notices, form fields, and product variation states on the frontend.

Troubleshooting:

- For missing sections, rows, or modules on phone/tablet, check Visibility options and responsive-specific layout settings first.
- For display issues after updating, clear caches, regenerate static CSS if available, inspect child-theme/custom CSS, verify plugin compatibility, then test with a minimal reproduction.
- For custom CSS issues, confirm where the CSS was added, whether selectors still match Divi 5 output, and whether a native option now replaces the CSS.
- For missing `Enable Divi Builder`, check theme/plugin state, post type support, permissions, and editor settings.

## Beginner Coaching Mode

When the user is at the controls, explain Divi in this mental model:

- WordPress stores the site, pages, posts, menus, media, users, and plugins.
- Divi 5 is the visual design system and builder used to assemble and style those WordPress items.
- A page is built from big containers to small content: Section -> Row -> Column -> Module.
- A Theme Builder template is a reusable layout rule for many pages, not just one page.
- A variable is a named design value.
- A preset is a saved style recipe.
- A loop is a repeated design connected to WordPress content.
- An interaction is a rule that says: when this happens, do that to this target.
- A canvas is hidden or separate content that can be opened, reused, or placed through a portal.

For hands-on coaching:

1. Say exactly where to click.
2. Name the panel, tab, and option group.
3. Explain the reason in one plain sentence.
4. Ask the user to save and verify after meaningful milestones.
5. When multiple choices exist, recommend the most native Divi 5 option first.

## Recommended Build Order

For new websites:

1. Gather brand assets, copy, images, sitemap, navigation, CTAs, forms, integrations, and reference sites.
2. Install/activate Divi 5 and create a staging-safe baseline.
3. Define Design Variables for colors, fonts, fluid type, spacing, radius, logos, business info, and key URLs.
4. Create Option Group Presets for typography, buttons, spacing, borders, backgrounds, fields, cards, and images.
5. Create Element Presets for common modules/components.
6. Build global header/footer in Theme Builder.
7. Build reusable page sections and save useful patterns to the Divi Library or Divi Cloud.
8. Build main pages, then archive/search/blog/product templates.
9. Add interactions/canvases only after the base content and responsive layout are sound.
10. Test desktop/tablet/phone, frontend/public page, forms, menus, accessibility basics, cache, and performance.

For existing Divi 4 sites:

1. Work on staging first.
2. Back up files and database.
3. Inventory plugins, child theme code, custom CSS, global modules, shortcodes, Theme Builder templates, and third-party Divi modules.
4. Check Elegant Themes compatibility guidance and known issues.
5. Convert/test in small batches when possible.
6. Rebuild brittle custom CSS as Divi 5 variables, presets, Flexbox/Grid, or Composable Settings when practical.
7. Verify frontend rendering, responsive visibility, menus, forms, WooCommerce, and dynamic templates before launch.
